# MeeTap - 用户指南

通过 macOS 内部音频（BlackHole）自动录制 Teams / Zoom / 飞书 / 腾讯会议，无需屏幕录制权限。

---

## 快速开始

```bash
meetap start    # 开会前或开会后，随时启动
meetap stop     # 开完后，自动保存 + 转录
```

录音文件保存在 `~/Record/MMDD_HHMM/` 目录下，录制结束后自动转录为文字。

### 使用须知

- **启动顺序不限** — 先 `meetap start` 再入会，或者先入会再 start，都可以
- **支持任意音频设备** — AirPods、有线耳机、Mac 扬声器均可，start 时自动检测
- **⚠️ 录制期间不要手动切换音频输出** — 这是唯一需要注意的事。meetap 会自动转发声音到你的耳机/扬声器，如果中途手动切换了输出设备，会导致录音无声。如果不小心切了，切回原设备即可恢复

## 首次使用

### 前提条件

以下软件需要已安装（正常情况下已装好）：

| 软件 | 用途 | 安装方式 |
|------|------|---------|
| BlackHole 2ch | 虚拟音频通道 | `brew install blackhole-2ch` |
| ffmpeg | 音频录制编码 | `brew install ffmpeg` |
| AWS CLI | 自动转录（AWS Transcribe） | `brew install awscli` |

### 一键配置会议 App

```bash
meetap setup
```

该命令会依次配置：
1. **Zoom** — 自动将扬声器改为 "Same as System"（全自动，无需操作）
2. **Teams** — 打开设备设置页面，手动选择 "BlackHole 2ch"
3. **腾讯会议** — 打开应用，引导手动在音频设置中选择 "BlackHole 2ch"

> 此配置只需执行一次。也可以单独配置：
> - `meetap setup-zoom`
> - `meetap setup-teams`
> - `meetap setup-wemeet`

### 飞书

飞书默认跟随系统输出，无需额外配置。

## 日常使用

### 开始录制

```bash
meetap start
```

脚本会自动完成以下操作：
1. 检测你当前使用的音频输出设备（AirPods、耳机、扬声器等）
2. 切换系统输出到 BlackHole
3. 启动音频转发（BlackHole → 你的耳机/扬声器）
4. 开始录制（系统音频 + 麦克风）

输出示例：
```
📍 当前音频输出: MacBook Pro Speakers
🔧 切换系统输出到 BlackHole 2ch ...
✅ 系统输出已切换到: BlackHole 2ch
🔧 启动音频转发: BlackHole → MacBook Pro Speakers ...
✅ 音频转发已启动 → MacBook Pro Speakers

🎙️  录制已开始
   文件: ~/Record/0417_1400/meeting_20260417_140000.m4a
   播放: MacBook Pro Speakers

   停止录制: meetap stop
```

### 停止录制

```bash
meetap stop
```

输出示例：
```
⏳ 等待录制完成...
🔊 音频输出已恢复: MacBook Pro Speakers

✅ 录制已保存
   文件: ~/Record/0417_1400/meeting_20260417_140000.m4a
   大小: 26.5 MB
   时长: 28分13秒

📝 后台转录已启动（完成后会收到通知）
```

转录在后台自动进行，不阻塞终端。完成后：
- 会话目录下生成 `transcript.txt`（带说话人标签）
- 自动调用 Claude Sonnet 4.6（Bedrock）生成结构化会议纪要 `meeting-notes.md`
- macOS 弹出通知提醒

```
~/Record/0417_1400/
├── meeting_20260417_140000.m4a   # 录音文件
├── transcript.txt                # 转录文本（自动生成）
├── meeting-notes.md              # 会议纪要（Claude 自动生成）
├── transcribe-raw.json           # Transcribe 原始结果
└── transcribe.log                # 转录过程日志
```

## 录音与转录文件

每次录制会在 `~/Record/` 下自动创建一个以日期时间命名的文件夹（如 `0417_1400`），所有相关文件集中存放：

```
~/Record/0417_1400/
├── meeting_20260417_140000.m4a   # 录音文件（AAC 128kbps 单声道，约 1MB/分钟）
├── transcript.txt                # 转录文本（带说话人标签，stop 后自动生成）
├── meeting-notes.md              # 会议纪要（Claude 自动生成，含摘要和待办）
├── transcribe-raw.json           # AWS Transcribe 原始 JSON 结果
└── transcribe.log                # 转录过程日志
```

如果录制停止后未产生录音文件，空文件夹会被自动删除。

## 录制内容说明

| 内容 | 是否录入 |
|------|---------|
| 对方说话的声音 | 是（通过系统音频捕获） |
| 你自己说话的声音 | 是（通过麦克风捕获） |
| 系统通知声、其他应用声音 | 是（录制期间所有系统音频都会被捕获） |
| 浏览器音频（视频、在线课程等） | 是（所有系统音频输出均会被捕获） |

> 建议戴耳机开会录制，避免麦克风拾取扬声器回声。

### 不只是会议

MeeTap 录制的是系统级音频输出，不限于会议应用。以下场景同样适用：

- **浏览器视频**：YouTube、在线培训、Webinar 回放
- **在线课程**：Coursera、Udemy 等平台的课程音频
- **播客 / 音频内容**：任何通过系统输出播放的音频

使用方式与录制会议完全一样：`meetap start` → 播放内容 → `meetap stop`，录制结束后自动转录并生成纪要。

> 注意：当前默认同时录制麦克风输入（你的声音）。如果只想录制系统音频（如纯视频回放），环境噪音也会被录入。

## 不同音频设备

不需要任何额外操作。每次执行 `start` 时，脚本会自动检测当前系统输出设备：

- 戴 AirPods 开会 → `start` 自动用 AirPods ✅
- 用有线耳机开会 → `start` 自动用有线耳机 ✅
- 在工位外放开会 → `start` 自动用 MacBook 扬声器 ✅（建议戴耳机，避免回声）

## 常见问题

### Q: 录制中耳机里听不到声音？

meetap 通过 audio-monitor 将声音转发到你的耳机/扬声器。如果听不到：
1. 检查 `meetap status`，确认音频转发是否在运行
2. 检查会议 App 扬声器设置是否为 "Same as System"（Zoom）或 "BlackHole 2ch"（Teams/腾讯会议）
3. 如果不是，重新执行 `meetap setup`

### Q: stop 后其他应用没声音？

脚本 stop 时会自动恢复音频输出。如果异常退出（如终端被关闭），手动恢复：

```bash
# 查看可用输出设备
~/bin/audio-multi-output list

# 手动切回，例如切到 MacBook 扬声器
~/bin/audio-multi-output set-default <设备ID>
```

或者直接在系统设置 → 声音 → 输出中选择你的耳机/扬声器。

### Q: 录音文件没有声音 / 只有开头几秒有声音？

最常见的原因：**录制期间手动切换了系统音频输出设备**。

`meetap start` 会将系统输出切到 BlackHole，同时通过 audio-monitor 转发声音到你的耳机/扬声器。如果录制中你在系统设置或菜单栏切换了输出设备，音频就不再经过 BlackHole，录音变成静音。

解决方法：**录制期间不要手动切换音频输出。如果不小心切了，切回原来的设备即可恢复。** 需要调整音量直接用键盘音量键即可。

### Q: 录音文件是空的或很小？

检查 ffmpeg 日志：
```bash
cat /tmp/meeting-record/ffmpeg.log
```

常见原因：BlackHole 未正确安装，或会议 App 的音频没有走系统输出。

### Q: 同时开了两个会议 App，能录到吗？

能。所有通过系统输出的音频都会被 BlackHole 捕获并混录到同一个文件中。

### Q: 转录没有生成 transcript.txt？

检查转录日志：
```bash
cat ~/Record/0417_1400/transcribe.log
```

常见原因：
- AWS CLI 未安装或未配置凭证（`aws sts get-caller-identity` 验证）
- 网络问题导致 S3 上传失败
- 录音文件过短或无有效音频

转录失败不影响录音文件保存，可稍后手动重试。

### Q: 重启电脑后需要重新配置吗？

不需要。Teams、Zoom、腾讯会议的扬声器设置都是持久化的。`start` 时会自动重建多输出设备。
