# 会议录制工具 - 安装说明

在新 Mac 上从零安装，实现 Zoom / Teams / 飞书 / 腾讯会议的音频录制。

---

## 系统要求

| 项目 | 要求 |
|------|------|
| macOS | 13 (Ventura) 或更高 |
| 芯片 | Intel / Apple Silicon 均可 |
| 磁盘 | 约 500MB（主要是 ffmpeg） |
| 权限 | 管理员权限（安装 Homebrew 和 BlackHole） |

---

## 一键安装

### 方式 1：拷贝整个项目目录后运行

```bash
# 假设你把项目文件拷贝到了 ~/meeting-record-install/
cd ~/meeting-record-install/
bash install.sh
```

### 方式 2：只有安装脚本和源文件

将以下文件放到同一目录：

```
install-dir/
├── install.sh
└── src/
    ├── audio-multi-output.swift
    ├── audio-monitor.swift
    ├── meeting-record.sh
    └── meeting-record-test.sh
```

然后运行：

```bash
cd install-dir/
bash install.sh
```

安装脚本会自动完成以下所有步骤。

## 手动安装（如果一键安装失败）

### 第 1 步：安装 Homebrew

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Apple Silicon Mac 安装后需要添加到 PATH：

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc
```

### 第 2 步：安装依赖

```bash
brew install blackhole-2ch ffmpeg switchaudio-osx
```

| 软件 | 用途 | 验证命令 |
|------|------|---------|
| blackhole-2ch | 虚拟音频通道（48kHz） | `brew list blackhole-2ch` |
| ffmpeg | 音频采集和 AAC 编码 | `ffmpeg -version` |
| switchaudio-osx | 命令行切换音频设备 | `SwitchAudioSource -c` |

### 第 3 步：安装 Xcode Command Line Tools

```bash
xcode-select --install
```

如果已安装会提示 "already installed"，可忽略。

### 第 4 步：创建目录

```bash
mkdir -p ~/bin ~/Record
```

### 第 5 步：复制源文件并编译

将 4 个源文件复制到 `~/bin/`：

```bash
cp audio-multi-output.swift ~/bin/
cp audio-monitor.swift ~/bin/
cp meeting-record.sh ~/bin/
cp meeting-record-test.sh ~/bin/
```

编译 Swift 工具：

```bash
swiftc -O -framework CoreAudio -framework AudioToolbox \
  ~/bin/audio-multi-output.swift -o ~/bin/audio-multi-output

swiftc -O -framework CoreAudio -framework AudioToolbox \
  ~/bin/audio-monitor.swift -o ~/bin/audio-monitor
```

设置可执行权限：

```bash
chmod +x ~/bin/meetap
chmod +x ~/bin/audio-multi-output ~/bin/audio-monitor
```

创建 Python 虚拟环境（用于会议纪要自动生成）：

```bash
python3 -m venv ~/bin/meetap-venv
~/bin/meetap-venv/bin/pip install boto3
```

> meetap-venv 用于调用 AWS Bedrock (Claude Sonnet 4.6) 自动生成结构化会议纪要。转录完成后自动触发，无需手动操作。

### 第 6 步：配置 PATH

```bash
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 第 7 步：验证安装

```bash
# 检查所有工具
audio-multi-output blackhole-uid     # 应输出 BlackHole 的 UID
audio-monitor --help                 # 应显示用法
ffmpeg -version                      # 应显示版本
SwitchAudioSource -c                 # 应显示当前设备名
```

或运行自动化测试：

```bash
meeting-record-test.sh
```

## 配置会议 App

### Zoom（全自动）

安装脚本会自动将 Zoom 扬声器设为 "Same as System"。

如果需要手动配置：

```bash
meeting-record.sh setup-zoom
```

或在 Zoom 中手动操作：
1. 设置 → Audio
2. Speaker → 选择 **Same as system**

### Teams（需手动点一下）

Teams 没有 "System Default" 选项，需要手动选择 BlackHole 2ch：

```bash
meeting-record.sh setup-teams
```

或手动操作：
1. 打开 Teams → 设置 → Devices
2. Speaker → 选择 **BlackHole 2ch**

> 只需设置一次。Teams 会记住这个选择。

### 腾讯会议（需手动点一下）

```bash
meeting-record.sh setup-wemeet
```

或手动操作：
1. 打开腾讯会议 → 点击头像 → 设置 → 音频
2. 扬声器 → 选择 **BlackHole 2ch**

> 只需设置一次。

### 飞书

无需任何配置。飞书默认跟随系统输出。

## 注意事项

### 安装相关

1. **BlackHole 安装后需重启**：部分 macOS 版本安装 BlackHole 后需要重启才能识别虚拟音频设备。如果 `audio-multi-output blackhole-uid` 报错找不到设备，请重启 Mac。

2. **安全性提示**：安装 BlackHole 时，macOS 可能弹出"系统扩展已阻止"提示。需要到 **系统设置 → 隐私与安全性** 中点击"允许"。

3. **Homebrew 安装慢**：如果在中国大陆，Homebrew 安装可能很慢。可以使用镜像源：
   ```bash
   export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git"
   export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git"
   ```

4. **ffmpeg 编译时间长**：首次安装 ffmpeg 可能需要 5-10 分钟（Homebrew 从源码编译）。

### 使用相关

5. **蓝牙耳机（AirPods）不支持录音**：蓝牙耳机采样率（24kHz）与 BlackHole（48kHz）不兼容。录音时请使用 MacBook 内置扬声器或有线耳机。

6. **录音期间不要手动切换音频设备**：录音期间系统输出被设为 BlackHole，如果手动切换会导致录音中断。需要切换请先执行 `meeting-record.sh stop`。

7. **异常退出的恢复**：如果脚本异常退出（如终端被关闭），系统输出可能还停在 BlackHole（此时所有声音都听不到）。恢复方法：
   ```bash
   # 方法 1：用命令恢复
   SwitchAudioSource -s "MacBook Pro Speakers" -t output

   # 方法 2：系统设置 → 声音 → 输出 → 选择你的扬声器
   ```

8. **重启后设备消失是正常的**：BlackHole 虚拟设备在系统重启后仍存在（它是内核扩展），但 `meeting-record.sh start` 创建的临时状态会丢失。直接重新 `start` 即可。

9. **多用户环境**：BlackHole 是系统级驱动，所有用户共享。但录音工具安装在各自的 `~/bin/` 下，互不干扰。

### Teams 特殊说明

10. **Teams 扬声器选 BlackHole 2ch 后不在录音时会没声音**：这是正常的。不录音时系统输出是 MacBook Speaker，但 Teams 输出到 BlackHole（没人在听）。解决方法：
    - 不录音时在 Teams 里手动切回 MacBook Pro Speakers
    - 或者始终通过 `meeting-record.sh start/stop` 管理录音（start 时系统输出切到 BlackHole，声音通过 audio-monitor 转发到扬声器）

11. **更好的 Teams 方案**：如果 Teams 支持选择 "Same as system"（某些版本可能有），优先选它，这样就不需要手动切换了。

## 卸载

```bash
# 删除工具
rm -f ~/bin/audio-multi-output ~/bin/audio-multi-output.swift
rm -f ~/bin/audio-monitor ~/bin/audio-monitor.swift
rm -f ~/bin/meeting-record.sh ~/bin/meeting-record-test.sh

# 删除运行时状态
rm -rf /tmp/meeting-record

# 删除依赖（可选）
brew uninstall blackhole-2ch
brew uninstall switchaudio-osx
# ffmpeg 可能被其他工具使用，谨慎卸载
# brew uninstall ffmpeg

# 删除录音文件（谨慎！）
# rm -rf ~/Record

# 清理 PATH（编辑 ~/.zshrc 删除 export PATH="$HOME/bin:$PATH"）
```

## 故障排查

| 问题 | 解决方案 |
|------|---------|
| `blackhole-uid` 报错 | 重启 Mac；检查系统设置是否阻止了系统扩展 |
| `swiftc` 找不到 | 运行 `xcode-select --install` |
| `SwitchAudioSource` 找不到 | 运行 `brew install switchaudio-osx` |
| 录音文件是静音的 | 检查会议 App 扬声器是否设为 BlackHole 2ch 或 Same as System |
| 录音时听不到声音 | 确认 audio-monitor 进程在运行：`meeting-record.sh status` |
| stop 后仍然没声音 | 手动切换：`SwitchAudioSource -s "MacBook Pro Speakers" -t output` |
| Teams 没声音（非录音时） | Teams 扬声器切回 MacBook Pro Speakers |
| 编译警告 `UnsafeMutableRawPointer` | 无害警告，可忽略 |

## 快速验证清单

安装完成后，按以下步骤验证：

- [ ] `audio-multi-output blackhole-uid` 输出 BlackHole UID
- [ ] `SwitchAudioSource -a -t output` 列表中包含 BlackHole 2ch
- [ ] `meeting-record.sh start` 成功启动（听到扬声器有声音）
- [ ] `meeting-record.sh stop` 正常停止（音频恢复到扬声器）
- [ ] 录音文件 `~/Record/meeting_*.m4a` 存在且有有效音频
- [ ] Zoom 设置 → Audio → Speaker 显示 "Same as system"
- [ ] Teams 设置 → Devices → Speaker 显示 "BlackHole 2ch"
